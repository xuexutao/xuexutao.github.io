视觉语言预训练(Vision-Language pre-training)旨在利用图像文本数据来教会模型共同理解视觉和文本信息的能力

尽管视觉语言预训练取得了巨大成功，但现有方法有两个主要局限性 ：

  • 从模型的角度来看， 大多数现有的预训练模型都不够灵活，无法适应广泛的视觉语言任务。基于编码器的模型不太容易直接转移到文本生成任务中，而编码器-解码器模型尚未成功用于图像-文本检索任务。
  • 从数据的角度来看，大多数模型都使用从 Web 自动收集的图像和替代文本对进行预训练。但是，Web 文本通常不能准确描述图像的视觉内容 ，使它们成为嘈杂的监督来源。

使用 BLIP 翻转脚本

Bootstrapping Language-Image Pre-training

  • 一种新的模型架构 ，支持比现有方法更广泛的下游任务
  • 一种新的数据集引导方法，用于从嘈杂的 Web 数据中学习

BLIP是编码器-解码器混合架构，由三个视觉语言目标联合预训练：图像文本的对比学习、图像文本匹配和图像条件语言建模。

1、Multimodal mixture of Encoder-Decoder，MED

颜色相同的块 共享权重

1.1、Unimodal encoder 单峰编码器

单独编码图像和文本。文本编码器与Bert相同，其中[Cls]令牌将附加到文本输入的开头以总结句子。

Image-Text Contrastive Loss (ITC)：激活单峰编码器

It aims to align the feature space of the visual transformer and the text transformer by encouraging positive image-text pairs to have similar representations in contrast to the negative pairs

引入了动量编码器来产生特征，并且从动量编码器中创建了软标签，作为训练目标，以说明负对中的潜在阳性

1.2、Image-grounded text encoder

从图可以看出，他在每一个text-encoder的transformers中self-attention和Feed Forward层之间插入了一个Cross-attention层。特定于任务的[Encoder]令牌附加到文本上，[Encode]的输出嵌入作为图像文本对的多模式表示

Image-Text Matching Loss (ITM)：它旨在学习图像文本多模式表示，以捕获视觉和语言之间的细粒度对齐。 ITM是一项二进制分类任务，模型使用ITM head（线性层）来预测图像文本对在给定其多模式特征情况下是positive（匹配）还是negative（不匹配）。为了找到更多信息的负面影响，我们采用了硬采矿策略（hard negative mining strategy）。在批处理中具有较高对比度相似性的负面配对更有可能选择计算损失

1.3、Image-grounded text decoder

从图可以看出，他用Causal self-attention层代替了bi-directional self-attention层。[Decoder]令牌用于指示序列的开始，使用末端令牌来表示其末端

Language Modeling Loss (LM)：该解码器旨在生成给定图像的文本描述。它优化了cross entropy loss，该损失训练模型以自回归方式最大化文本的可能性。计算损失时，应用0.1的标签平滑（label smooth）。与VLP中广泛使用的MLM损失相比，LM具有具有概括能力的模型，可以将视觉信息转换为连贯的字幕。

为了在利用多任务学习时执行有效的预训练，文本编码器和文本解码器共享除SA层以外的所有参数。原因是SA层最好捕获编码和解码任务之间的差异。特别是，编码器采用双向自我注意力来为当前的输入令牌建立表示形式，而解码器则采用因果自我注意力来预测隔壁令牌。另一方面，嵌入层，CA层和FFN在编码和解码任务之间的功能相似，因此共享这些层可以提高训练效率，同时受益于多任务学习，

2、Captioning and Filtering，CapFilt

高效率利用噪声网络数据

Captioner and Filter都是从MED pre-training model经过微调得到的！！

字幕器 Captioner：给一张网络图片，生成字幕。它是一个视觉文本解码器，在 COCO 数据集上使用 LM 目标函数微调。给定网络图片 ，Captioner 生成字幕

过滤器 Filter：过滤掉噪声图文对。它是一个视觉文本编码器，看文本是否与图像匹配，在 COCO 数据集上使用 ITC 和 ITM 目标函数微调。Filter 删除原始 Web 文本 和合成文本 中的嘈杂文本，如果 ITM 头将其预测为与图像不匹配，则认为文本有噪声

最后，将过滤后的图像-文本对与人工注释相结合，形成一个新的数据集，重新用它来预训练一个新的模型，即为最后的model
